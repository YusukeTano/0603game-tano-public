### 1\. 概要（詳細版）

本セクションは、ゲーム企画「Project: "Void Rusher"」の核心をなす基本情報です。開発者や関係者がプロジェクトの全体像と方向性を一目で理解することを目的とします。

#### **ゲームタイトル（仮）: Project: "Void Rusher"**

  * **詳細:**
      * これは開発中のコードネームです。"Void"は「虚空」「宇宙」というゲームの舞台を示唆し、"Rusher"は「突撃するもの」を意味します。無数の敵をなぎ倒す爽快なゲームプレイと、敵が絶え間なく殺到してくるサバイバル感を表現しています。
      * 将来的には、よりキャッチーで覚えやすい正式名称を検討しますが、現時点ではこの仮タイトルがプロジェクトの目指す方向性を明確に示します。

#### **ジャンル: 全方位スクロール・サバイバーライクアクション**

  * **詳細:**
      * **全方位スクロール:** プレイヤーは上下左右360°に自由に移動できる広大なマップ（戦闘空域）を舞台とします。閉鎖空間ではなく、無限の広がりを感じさせるステージ設計が特徴です。
      * **サバイバーライク:** 近年人気のジャンルであり、主な特徴は「攻撃の自動化」「敵の殲滅による経験値獲得とレベルアップ」「レベルアップ時に提示されるランダムなスキル選択による自己強化（ビルド）」です。これにより、プレイヤーは複雑な攻撃操作から解放され、敵の弾幕を避ける「回避行動」と、スキルを選ぶ「戦略的判断」に集中できます。
      * **アクション:** 敵の攻撃をリアルタイムで回避し、有利な位置取りを模索するなど、プレイヤーの反射神経と瞬時の判断力が求められる、アクション性の高いゲームです。

#### **プラットフォーム: 単一のHTMLファイルで動作するPC Webブラウザ**

  * **詳細:**
      * 本作は、PCのWebブラウザ（Google Chrome, Firefox, Safari, Edgeなど）上で動作します。専用アプリケーションのダウンロードやインストールは一切不要です。
      * 最大の技術的特徴は\*\*「単一HTMLファイル」\*\*での完結です。通常、Webコンテンツは複数のファイル（HTML, CSS, JavaScript, 画像, 音声など）で構成されますが、本作ではそれら全ての要素を一つのHTMLファイルに内包します。これにより、ファイルの配布や管理が劇的に容易になり、「いつでも、どこでも、手軽に遊べる」というコンセプトを技術的に実現します。

#### **コンセプト: 広大な宇宙を舞台に、個性的な機体を選択し、無数の敵をなぎ倒す爽快感と、自らを強化していく達成感をリズミカルなBGMと共に味わう、近未来的トップダウンアクションゲーム。**

  * **詳細:**
      * 本作がプレイヤーに提供したい\*\*中核的な体験（コアエクスペリエンス）\*\*を言語化したものです。
      * **「広大な宇宙」:** 世界観のテーマです。
      * **「個性的な機体を選択し」:** リプレイ性（繰り返し遊ぶ楽しさ）とプレイヤーの自己表現を促す要素です。
      * **「無数の敵をなぎ倒す爽快感」:** ゲームが提供する最も重要な感情的報酬です。派手なエフェクトや効果音でこれを最大化します。
      * **「自らを強化していく達成感」:** サバイバーライクの面白さの根幹です。最初は弱かった自機が、プレイヤーのスキル選択によって飛躍的に強力になっていく成長体験を提供します。
      * **「リズミカルなBGMと共に味わう」:** BGMを単なる背景音楽ではなく、ゲームプレイのテンションやリズムと一体化させ、相乗効果で爽快感を増幅させる重要な演出要素と位置づけます。
      * **「近未来的トップダウンアクション」:** SF的なビジュアルと、キャラクターを真上から見下ろす視点のゲームであることを示します。

#### **ターゲット層: 手軽に爽快感を味わいたいライト～ミドルコアゲーマー。SFやサイバーパンク、可愛い系の世界観が好きなユーザー。**

  * **詳細:**
      * 本作を**誰に届けたいか**を定義します。
      * **「手軽に爽快感を味わいたいライト～ミドルコアゲーマー」:** 1プレイが数分～十数分で完結し、複雑な操作を覚えなくても直感的に楽しめるため、普段あまりゲームをしない層から、ある程度の遊びごたえを求める層まで幅広くアピールします。
      * **「SFやサイバーパンク、可愛い系の世界観が好きなユーザー」:** ゲームのビジュアルデザインの方向性を示します。クールな戦闘機"Strider-V"と、キュートなドローン"Pico-Rabbit"という対照的な機体を用意することで、異なる美的センスを持つプレイヤー双方にリーチします。

### 2\. ゲームフローと画面遷移（詳細版）

本セクションでは、プレイヤーがゲームを起動してから終了するまでの一連の流れ（フロー）と、各場面の切り替わり（画面遷移）を具体的に定義し、ゲーム全体の構造とユーザー体験を明確にします。

#### **1. タイトル画面 (エントリーポイント)**

  * **目的:** ゲームの第一印象を決定づけ、プレイヤーを世界観に引き込む導入画面。ゲーム開始を促す。
  * **画面要素:**
      * **ゲームロゴ:** 画面中央に大きく配置された、ゲームの顔。
      * **「START」ボタン:** ゲームを開始するための唯一のアクションボタン。点滅や微細な拡縮アニメーションでプレイヤーの注意を引く。
      * **背景:** ゲームの舞台となる、動的な宇宙空間。
      * **BGM:** これから始まる冒険への期待感を煽る、印象的なテーマ曲。
      * **クレジット表記:** 画面隅に、開発者名や著作権情報を小さく表示。
  * **遷移:** 「START」ボタンのクリックで**キャラクター選択画面**へ遷移。

#### **2. キャラクター選択画面 (プレイ準備)**

  * **目的:** プレイヤーに操作キャラクターを選択させ、リプレイ性（繰り返し遊ぶ楽しさ）を提供。各キャラクターの個性と性能差を提示し、戦略的思考を促す。
  * **画面要素:**
      * **キャラクターリスト:** 選択可能な全キャラクター（"Strider-V", "Pico-Rabbit"等）のビジュアルを横一列に表示。選択中のキャラクターはハイライトされ、中央で待機モーションを行う。
      * **キャラクター情報:** 選択中のキャラクターの「機体名」「コンセプト」「基本性能の特徴」などを表示。
      * **「決定」ボタン:** 選択を確定し、ゲームプレイを開始するボタン。
      * **BGM:** タイトル画面の曲調を引き継ぎつつ、これから始まるプレイへの高揚感を高めるリズミカルなアレンジに変化。
  * **遷移:** キャラクターを選択し「決定」ボタンをクリックすると、**ゲームプレイ画面**へ遷移。選択されたキャラクター情報はシステムに保持される。

#### **3. ゲームプレイ画面 (メインループ)**

  * **目的:** ゲームの中核となる実際のプレイ画面。プレイヤーは「回避」と「スキル選択による強化」を繰り返し、ハイスコアを目指す。
  * **画面要素:**
      * **ゲーム空間:** プレイヤー機、無数の敵、アイテム、エフェクトなどが表示されるメインエリア。
      * **HUD（ヘッドアップディスプレイ）:** HP、レベル、スコアなどの重要情報を画面四隅に常時表示。
      * **ミニマップ:** 広大な戦闘空域の全体像と現在位置を表示。
      * **一時停止ボタン:** 画面の隅に配置。
      * **BGM/SE:** 状況に応じてダイナミックに変化するBGMと、爽快感を高める効果音。
  * **遷移:**
      * プレイヤーのHPが0になると**ゲームオーバー画面**へ遷移。
      * 「一時停止ボタン」をクリックすると**一時停止メニュー**が表示される。

#### **4. 一時停止メニュー (中断機能)**

  * **目的:** プレイヤーが一時的にゲームを中断し、休憩や設定変更（将来的な拡張）を行えるようにする。
  * **画面要素:**
      * **オーバーレイ:** ゲームプレイ画面の上に半透明のフィルターがかかり、背景のゲームは完全に静止する。
      * **メニューパネル:** 画面中央に表示。
      * **「ゲームに戻る」ボタン:** メニューを閉じ、即座にゲームプレイを再開。
      * **「タイトルへ戻る」ボタン:** 現在のプレイを破棄し、タイトル画面へ戻るか確認後、**タイトル画面**へ遷移。
      * **BGM:** 静かなアンビエントな曲調に切り替わり、クールダウンした雰囲気を提供。
  * **遷移:**
      * 「ゲームに戻る」ボタンで**ゲームプレイ画面**に復帰。
      * 「タイトルへ戻る」ボタンで（確認後）**タイトル画面**へ遷移。

#### **5. ゲームオーバー画面 (結果表示)**

  * **目的:** 1プレイの終了を伝え、結果（スコア等）をフィードバックする。再挑戦やゲーム終了を促す。
  * **画面要素:**
      * **「GAME OVER」表示:** 大きなテキストで表示。
      * **リザルトパネル:** 最終スコア、生存時間、総討伐数などを表示（将来的にハイスコア表示も検討）。
      * **「リトライ」ボタン:** スコア等をリセットし、キャラクター選択画面からプレイをやり直す。
      * **「タイトルへ戻る」ボタン:** タイトル画面へ戻る。
      * **BGM:** 挑戦の終わりを告げる、静かで悲壮感のある専用曲。
  * **遷移:**
      * 「リトライ」ボタンで**キャラクター選択画面**へ遷移。
      * 「タイトルへ戻る」ボタンで**タイトル画面**へ遷移。

### 3\. ファイル構成（詳細版）

本セクションでは、ゲームのファイル構造を定義します。本作最大の技術的特徴は、究極のシンプルさとポータビリティを実現する\*\*「単一ファイル構成」\*\*です。

#### **基本方針: すべてを一つのHTMLファイルに**

通常のWebコンテンツが複数のファイル（HTML, CSS, JS, 画像, 音声）で構成されるのに対し、本作ではこれら全ての要素を **`index.html`** という**たった一つのファイル**に集約します。

  * **配布の容易さ:** `index.html` ファイル一つをコピーするだけで配布が完結。ファイルの不足やパスの間違いといったトラブルを根絶します。
  * **管理のシンプルさ:** 関連ファイルが一つにまとまっているため、バージョン管理やバックアップが極めて容易です。
  * **オフライン動作:** 一度ファイルをダウンロードすれば、インターネット接続がない環境でもブラウザで開くだけでプレイ可能です。

#### **`index.html` の内部構造**

単一ファイルは、内部で以下の主要なセクションに分かれています。

##### **1. HTML (HyperText Markup Language) セクション**

  * **役割:** ゲームの「骨格」。Webページとしての基本的な構造を定義します。
  * **主な要素:**
      * `<head>`: ページのタイトル、文字コード指定、後述するCSSの記述エリア。
      * `<body>`: ブラウザに表示される内容の記述エリア。
          * **`<canvas id="game-canvas">`**: 本作の心臓部。全てのゲームグラフィックス（プレイヤー、敵、エフェクト等）は、この「キャンバス」要素の上にJavaScriptによってリアルタイムで描画されます。

##### **2. CSS (Cascading Style Sheets) セクション**

  * **役割:** ゲームの「見た目」。HTML要素のレイアウト、色、フォント等を装飾します。
  * **記述場所:** `<head>`タグ内に`<style>`タグを設置し、全てのCSSコードを直接記述します（外部ファイル化しない）。
  * **主な役割:**
      * ページ全体の背景色やCanvas要素の画面中央配置。
      * ボタンやUIパネルのスタイル定義（色、形、マウスオーバー時の変化など）。
      * スコア表示などのテキストのフォント、サイズ、色の指定。

##### **3. JavaScript セクション**

  * **役割:** ゲームの「魂」であり「頭脳」。全ての動き、ルール、計算を制御します。
  * **記述場所:** `<body>`タグの最後に`<script>`タグを設置し、全てのJavaScriptコードを直接記述します（外部ファイル化しない）。
  * **主な役割:**
      * **ゲームループ:** 毎秒60回画面を更新し続ける、ゲームの基本処理。
      * **描画処理:** Canvas APIを使い、キャラクターやエフェクトを`<canvas>`に描画。
      * **ロジック:** プレイヤー操作、敵AI、当たり判定、スコア計算、レベルアップ処理など、ゲームの全ルール。
      * **音声制御:** Web Audio APIを使い、BGMの再生や効果音の動的生成。

##### **4. アセット（画像・音声）の埋め込み**

  * **役割:** 通常は外部ファイルである画像や音声を、テキストデータとしてHTMLファイル内に直接埋め込みます。
  * **実現方法:**
      * **画像:** 一切使用しません。全てのビジュアルはCanvasの描画命令でリアルタイムに生成します。
      * **音声:** BGMなどの音声ファイルは、**データURI (Data URI)** スキームを利用します。音声データをBase64形式の長い文字列に変換し、JavaScriptコード内に直接記述することで、外部ファイルを読み込むことなく音声を再生可能にします。効果音はWeb Audio APIによりプログラムで直接生成するため、埋め込みは不要です。

### 4\. デザインコンセプト（詳細版）

本セクションでは、ゲームがプレイヤーに提供する視覚的・感覚的体験の設計思想とアートの方向性を定義します。

#### **全体テーマ: "Procedural Mecha & Minimal" (プロシージャルメカ & ミニマル)**

  * **Procedural (プロシージャル):** ゲームビジュアルは、用意された画像を貼り付けるのではなく、**全てコードによる計算（手続き）によってリアルタイムに生成**します。これにより、シャープで解像度に依存しないベクターアート調の質感と、滑らかなアニメーションを実現します。これは開発上の制約であると同時に、本作独自のスタイルを確立する核となります。
  * **Mecha (メカ):** 登場キャラクターは、複数のパーツで構成される「機械」としてのディテールと説得力を持たせ、プレイヤーが操作するのは魂のこもった「機体」であると感じさせます。
  * **Minimal (ミニマル):** 情報過多を避け、洗練されたシンプルな美しさを追求します。UIや背景は機能的で主張しすぎず、プレイヤー、敵、弾幕、エフェクトといったゲームプレイに直結する要素を際立たせます。

-----

#### 各要素の詳細

| 要素 | 詳細な説明 |
| :--- | :--- |
| **アートスタイル** | \*\*「闇とネオンのハイコントラスト」\*\*をテーマとします。背景は深い紺や黒を基調とし、その暗闇の中でプレイヤーや敵が放つ鮮やかなネオンカラー（シアン、マゼンタ等）が強烈な光を放ちます。この高いコントラストが視認性を極限まで高めると同時に、サイバーパンクを彷彿とさせるスタイリッシュでレトロフューチャーな雰囲気を醸し出します。全ての描画はCanvasの描画命令と発光効果(`shadowBlur`)を駆使して構築します。 |
| **UI/UXデザイン** | \*\*「没入感を阻害しない、機能的な美しさ」\*\*を目指します。UI要素は半透明の黒いパネルをベースにしたフラットデザインを採用。テキストは可読性の高いモダンなゴシック体を白や明るい色で表示し、暗い背景でも瞬時に読み取れるよう配慮します。ボタン等の操作要素は、マウスオーバー時に発光するなど、直感的なフィードバックを返します。画面遷移にはスムーズなフェードアニメーションを取り入れます。 |
| **キャラクター** | \*\*「シルエットで語る個性」\*\*を重視します。操作機体や敵は、役割や個性が一目で分かる特徴的なシルエットでデザインします。例えば、プレイヤー機はシャープな戦闘機や愛らしい動物型、敵は役割（高速型、重装甲型など）に応じた形状を持ちます。これにより、大量の敵が出現してもプレイヤーは瞬時に脅威度を判断し、戦略を立てることができます。 |
| **背景** | \*\*「静かなる躍動感」**を表現します。背景は複数のレイヤー（遠景の星々、中景の星雲など）で構成し、それぞれ異なる速度でスクロールさせる**「パララックス効果」\*\*を実装。これにより、プレイヤーは自分が広大な宇宙空間を高速で移動しているというダイナミックな感覚と、空間の深い奥行きを体感できます。 |

### 5\. ゲームシステム（詳細版）

本セクションでは、ゲームの根幹をなすルールやメカニクスを定義します。プレイヤーがどのようにゲームと関わり、どのような体験をするかの設計図です。

#### 5.1. ゲームステージ

  * **構造: 「戦闘空域（アリーナ）」**

      * **詳細:** ゲームは壁のない無限空間ではなく、画面の数十倍の広さを持つ有限の\*\*「戦闘空域（アリーナ）」\*\*で展開されます。これにより、プレイヤーは敵から逃げるための十分なスペースと開放感を得られます。同時に、完全に逃げ切れないという制約が緊張感を保ち、広さを利用して敵を誘導するなどの戦略的な立ち回りを促します。

  * **境界線: 「デッドリー・ゾーン」**

      * **詳細:** 戦闘空域の果ては、高エネルギーのプラズマが渦巻く壁など、SF的なビジュアルの\*\*「デッドリー・ゾーン」\*\*で区切られています。機体がこのゾーンに触れると、警告と共に継続的なダメージを受けます。これにより、プレイヤーは境界線を常に意識し、空間全体を使った戦略的な位置取りを管理する必要が生まれます。

#### 5.2. 操作方法

| 項目 | 詳細 |
| :--- | :--- |
| **移動** | プレイヤー機は常にマウスカーソルの位置を目指して移動します。キーボードは不要で、マウス一つで全ての移動操作が完結するため、誰でも直感的に操作可能です。 |
| **速度制御** | 移動速度は、**「機体自身の位置」と「マウスカーソルの位置」との距離**に応じて変化します。マウスを大きく動かすと機体は高速で追従し、ゆっくり動かせば減速します。 |
| **操作感** | この仕組みにより、プレイヤーの「速く/ゆっくり動きたい」という意思がマウス操作と直結。敵弾幕を素早く駆け抜けたり、敵の密集地帯を繊細に避けたりといった、レスポンスの良い\*\*「操縦している感」\*\*の高い体験を提供します。 |

#### 5.3. 攻撃システム

  * **自動攻撃:** プレイヤーは攻撃ボタンを押す必要がありません。装備中の武器が設定された間隔で**自動的**に攻撃を行います。これにより、プレイヤーは攻撃操作から解放され、敵の攻撃を避ける\*\*「回避」\*\*に完全に集中できます。
  * **照準と機体の向き:** プレイヤー機は、**常に最も近い敵の方向を自動的に向き続けます**（自動ロックオン）。これにより、マウス操作で後退しながらも機体は前方の敵を攻撃し続けるといった、映画的な戦闘シーンが自然に生まれます。

#### 5.4. レベルアップとスキルシステム

  * **成長ループ:** 敵がドロップする「経験値ジェム」を取得すると経験値ゲージが溜まり、満タンになると\*\*「レベルアップ」\*\*します。
  * **スキル選択:** レベルアップすると時間が停止し、3枚の「スキルカード」が表示されます。プレイヤーは、提示された能力（新武器の追加、既存武器の強化、移動速度アップ等）の中から**1つだけ**を選択します。この選択が、そのプレイにおける戦闘スタイル（ビルド）を方向づける重要な戦略的判断となります。
  * **ビルドの多様性:** 提示されるスキルはランダム性を含むため、プレイごとに異なるビルドを試せます。「範囲攻撃ビルド」「高耐久ビルド」「高速回収ビルド」など、プレイヤーの選択次第で無限の組み合わせが生まれ、高いリプレイ性を実現します。
  * **スキル例:**
      * **攻撃系:** 「ドローン追加」「ミサイル発射」「レーザー貫通化」「攻撃範囲拡大」
      * **防御系:** 「最大HP増加」「ダメージ軽減アーマー」「HP自動回復」
      * **補助系:** 「移動速度アップ」「経験値取得量アップ」「アイテム吸引範囲拡大」

### 6\. UI/UX (ユーザーインターフェース / ユーザーエクスペリエンス)（詳細版）

本セクションでは、プレイヤーが快適かつ直感的に遊ぶための情報設計と操作体験を定義します。「没入感を高め、ストレスをなくす」という目的のもと、UI（見た目の設計）とUX（心地よい体験）をデザインします。

#### 6.1. HUD (ヘッドアップディスプレイ)

ゲームプレイ中に視線を大きく動かすことなく、必要な情報を瞬時に把握できるよう、画面四隅に情報を分散配置します。

  * **画面上部（経過時間、レベル、経験値ゲージ、討伐数、スコア）:**
      * **目的:** プレイヤーの現在の進行度と成果を客観的に示すエリア。「経過時間」は難易度の指標としても機能します。
      * **デザイン:** 経験値ゲージはジェム取得のたびに滑らかに増加し、レベルアップ時には閃光を放つ演出で達成感を高めます。
  * **画面左下（プレイヤーのHPバー）:**
      * **目的:** 最も重要な生存指標であるHPを、視界の端で常に確認できるよう配置。
      * **デザイン:** HP残量に応じて色が（例: 青→黄→赤）と変化し、直感的に危険度を伝えます。被弾時には揺れるアニメーションでインパクトをフィードバックします。
  * **画面右下（現在取得しているスキルのアイコン一覧）:**
      * **目的:** 現在の自身の「ビルド（能力構成）」を一目で確認できるようにし、次のスキル選択の戦略を立てやすくします。
      * **デザイン:** 各スキルを象徴するシンプルなアイコンで表示。マウスカーソルを乗せると詳細な説明がポップアップします。

#### 6.2. ミニマップ

  * **配置と目的:** 画面右上に配置し、常に戦場全体の状況を把握できるようにします。敵の集団から逃げる方向の判断、アイテム位置の確認、危険な「デッドリー・ゾーン」との距離計測など、マクロな視点での戦略を可能にします。
  * **表示要素とデザイン:**
      * **プレイヤー:** 常にマップの中心に表示。
      * **敵（赤い点）:** 画面外の敵の位置や密集度も把握可能。ボスは大きく不吉なアイコンで表示。
      * **アイテム（アイコン）:** 回復アイテムなどの位置を逃さず確認。
      * **境界線:** ミニマップの端に警告ラインを表示。プレイヤーが境界に近づくと赤く点滅し、警告音と共に危険を知らせます。

#### 6.3. ゲーム内ボタンとメニュー

  * **設計思想:** 全てのメニュー画面は、半透明の黒いオーバーレイをゲーム画面に重ね、世界観の連続性を保ちつつ、プレイから一時的に離脱していることを示します。ボタンは明確なテキストラベルを持ち、選択時には色が変化するなど、直感的な操作が可能です。
  * **メニューごとの詳細:**
      * **タイトル画面:** ロゴと「START」ボタンのみのシンプルな構成でプレイヤーを迷わせません。
      * **キャラクター選択画面:** 機体を選択するとハイライトされ、起動アニメーションとSEが鳴るなど、選択する楽しさを演出します。
      * **一時停止メニュー:** 「ゲームに戻る」「タイトルへ戻る」という必要最低限の選択肢に絞り、迅速な判断を促します。
      * **ゲームオーバー画面:** 「リトライ」ボタンを目立たせることで、「もう一度挑戦しよう」という気持ちを自然に引き出し、ゲームの継続率を高めます。

### 7\. 音楽・サウンドデザイン（詳細版）

本セクションでは、ゲームの聴覚体験を設計します。サウンドは単なる背景ではなく、「爽快感」「リズム感」「達成感」をプレイヤーに直接伝える、最も重要な演出要素の一つと位置づけます。

#### 7.1. BGM (背景音楽)

BGMはプレイヤーの感情をリードし、ゲームプレイの展開をドラマチックに彩ります。状況に応じてシームレスに変化し、没入感と高揚感をコントロールします。

| 状況 | ジャンル/テーマ | 詳細な楽曲デザインと演出の方向性 |
| :--- | :--- | :--- |
| **通常プレイ時** | **シンセウェイヴ / ダークシンセ** | **体験:** 徐々に強くなる感覚と、迫る脅威の高まりを音楽で同期させます。\<br\>**構成:** ゲーム開始直後はBPM120-140程度のミニマルなビートで開始。レベルアップや時間経過と共にシンセのレイヤーが追加され、音楽も豪華で高揚感のあるものへとビルドアップしていきます。 |
| **ピンチ時** | **アグレッシブ・シンセウェイヴ** | **体験:** 心臓の鼓動が速くなるような、生理的な緊張感と焦りを演出します。\<br\>**構成:** HP低下時などにBGMのキーが上がり、心臓が脈打つような強いビート感を強調するサイドチェイン効果を深くかけることで、グルーヴと焦燥感を生み出します。 |
| **ボス戦** | **インダストリアル / EBM** | **体験:** 巨大な機械的脅威と対峙する、圧倒的なプレッシャーと闘争心をかき立てます。\<br\>**構成:** それまでのメロディアスな雰囲気から一変。歪んだ重いキックドラムと金属的なパーカッションが支配する、攻撃的で無機質なビートへと切り替わります。 |
| **スキル選択時** | **アンビエント / ドローン** | **体験:** 激しい戦闘から切り離された、時間が止まったかのような静かでクールな思考空間を提供します。\<br\>**構成:** レベルアップのファンファーレ後、全てのビートは消え、深いリバーブのかかった静謐なシンセパッド音のみが静かに流れます。これによりプレイヤーはクールダウンし、次のスキルを冷静に選べます。 |
| **ゲームオーバー時** | **ダーク・アンビエント** | **体験:** 挑戦の終わりを告げる、静かな絶望感と余韻。\<br\>**構成:** 機体破壊と同時に音楽が中断。その後、悲壮感のあるマイナーコードのシンセパッドが静かに流れ、ゆっくりと無音へとフェードアウト。「次こそは」という再挑戦への動機を促します。 |

#### 7.2. SE (効果音)

SEはプレイヤーのアクションに心地よいフィードバックを与え、爽快感に直結します。Web Audio APIによる音声合成を前提にデザインします。

| アクション | サウンドデザインの方向性（Web Audio APIによる生成例） |
| :--- | :--- |
| **プレイヤーの攻撃音** | **体験:** シャープで強力に感じさせる、歯切れの良いサウンド。\<br\>**構成:** 「ピュン！」というレーザー音。倍音を多く含むノコギリ波のピッチを瞬時に急降下させて鋭さを表現します。 |
| **敵へのヒット音** | **体験:** 攻撃が当たっていることを明確に伝え、連続ヒットが快感になるサウンド。\<br\>**構成:** 「ﾋﾟｯ」というデジタルな矩形波のサウンド。音の長さを極端に短くし、連続で鳴ってもリズミカルな快感を生み出します。 |
| **敵の破壊音** | **体験:** 敵を倒した行為への、最も直接的で最大の報酬。重量感と爽快感を両立。\<br\>**構成:** 「ｼｭﾜｯ\!/ドゥン」という破壊音。ホワイトノイズで霧散する「爽快感」を、低周波のサイン波で破壊の「重み」を同時に表現。この高低音の組み合わせが満足感を生みます。 |
| **レベルアップ音** | **体験:** 努力が報われたことを祝福する、高揚感に満ちたファンファーレ。\<br\>**構成:** 「ファーーーン！」というファンファーレ。柔らかい三角波を使い、上昇系の明るいアルペジオを奏でます。深いリバーブで空間に広がり、最高の達成感を演出します。 |
| **経験値ジェム取得音** | **体験:** 小さな成功体験の積み重ねが楽しくなる、収集欲を刺激するサウンド。\<br\>**構成:** 「ﾋﾟｺﾝ」という可愛らしい三角波のサウンド。連続取得時には音程が「ドレミファ」と上昇していき、大量に集める行為そのものが音楽的で楽しい体験になります。 |
| **被ダメージ音** | **体験:** 危険を瞬時に、そして明確に伝える警告音。\<br\>**構成:** 「ブズッ！」という不快なダメージ音。ディストーションをかけたノイジーな音で、本能的に「避けたい」と思わせます。他の音よりも目立つように調整し、注意を強制的に引きつけます。 |

### 9\. 開発上の留意点（詳細版）

本プロジェクトを円滑かつ高品質に完成させるため、開発者が特に注意すべき技術的課題と、その解決策を示します。

#### **1. ファイルサイズの管理**

  * **課題:** 全てのコードとアセットを単一ファイルに集約するため、対策を怠るとファイルサイズが肥大化し、初回ロード時間が長くなる危険性があります。
  * **対策:**
      * **BGMの最適化:** ファイルサイズ最大の要因であるBGMは、15～30秒程度の短いループ素材を高圧縮フォーマット（MP3, Opus等）で作成し、ゲーム内で再生し続けます。
      * **コードの圧縮 (Minify):** 公開時には、専用ツールでコードから不要な空白、改行、コメントを全て削除し、変数名を短縮する「圧縮」を行い、コードサイズを大幅に削減します。
      * **目標:** 快適なロード体験のため、最終的な`index.html`のファイルサイズを**2MB以下**に収めることを目標とします。

#### **2. コードの可読性**

  * **課題:** 数千行に及ぶコードが単一ファイルに混在するため、無秩序な記述は「スパゲッティコード」を招き、開発効率を著しく低下させます。
  * **対策:**
      * **論理的な分離:** `<style>`や`<script>`タグの前後には`// ------ GAME LOGIC ------ //`のような明確なコメントブロックを配置し、セクションを区切ります。
      * **モジュール設計:** ファイルは一つでも、`class Player { ... }`や`class EnemyManager { ... }`のようにクラス構文を積極的に利用し、機能ごとにコードを構造化します。
      * **命名規則の徹底:** 変数名や関数名には、`isGameOver`や`calculateEnemyPositions()`のように、役割が一目でわかる明確で一貫性のある命名規則を適用します。
      * **詳細なコメント:** 「何をしているか」だけでなく、「なぜこの処理が必要なのか」という意図をコメントとして残し、ロジックの可読性を担保します。

#### **3. パフォーマンス**

  * **課題:** ゲームエンジンの最適化の恩恵を受けられないため、大量のオブジェクトを描画する際に処理が重くなり、フレームレート（FPS）が低下するリスクがあります。
  * **対策:**
      * **オブジェクトプーリングの実装 (最重要):** 弾や敵など、大量に生成・消滅するオブジェクトを、その都度生成・破棄するのではなく、あらかじめ一定数をメモリ上に確保し「使い回す」仕組みを実装します。これにより、処理負荷を劇的に減らし、動作を安定させます。
      * **効率的なレンダリング:** 画面に見えていないオブジェクトの描画処理をスキップします。また、静的な背景などは一度見えないCanvas（オフスクリーンCanvas）に描画し、その結果を転写することで描画負荷を削減します。
      * **ゲームループの最適化:** ゲームのメインループには`setInterval`ではなく、ブラウザの描画タイミングに最適化された`requestAnimationFrame`を必ず使用し、滑らかなアニメーションを実現します。

### 10\. デザインアセット（詳細版）

本セクションでは、ゲームに登場する全ての動的オブジェクトの具体的なデザインを定義します。全てのアセットはCanvas 2D APIによるプロシージャル描画で生成されます。

#### 10.1. プレイアブルキャラクター

### 10.1.1. 先進型迎撃機 "Strider-V" (詳細)

#### **コンセプト**
**「戦場を支配する、洗練された孤高のエース」**
"Strider-V"は、無駄を一切削ぎ落とした機能美と、圧倒的な攻撃性能を体現する機体です。プレイヤーには、熟練のエースパイロットとして、その卓越した操縦技術で敵の大群を切り裂き、戦況を支配するようなエリート体験を提供します。デザインの根底にあるのは「冷徹な精密機械」という思想であり、その一挙手一投足がシャープで、計算され尽くしたものであることを感じさせます。

#### **デザイン詳細**
* **基本形状:** 全体は、鋭角な三角形で構成された**矢じり（アローヘッド）型**のシルエットを持ち、高速戦闘を主眼に置いた空力特性を視覚的に表現します。
    * **胴体 (Fuselage):** チタン合金を思わせる質感の、平たくシャープな機体です。表面には、装甲の継ぎ目を示す細いパネルラインが幾何学的に刻まれ、機械としての密度感を高めています。
    * **主翼 (Main Wings):** 緩やかなカーブを描く未来的な**後退翼**が、胴体の中ほどから接続されています。この翼は単なる飾りではなく、後述する強化に応じて形状が変化するギミックを持ちます。
    * **カナード翼 (Canards):** 機首のすぐ後ろには、小型の補助翼（カナード）が配置されており、高い機動性を持つ先進的な戦闘機であることを示唆します。
    * **推進ノズル:** 機体後部の中央には、強力な推力を生み出すことを想起させる、台形型のメインスラスターノズルが配置されています。

* **基本カラー:**
    * **本体:** 無塗装の軍用機を思わせる、硬質でメタリックな印象の銀色 (`#C0C0C0`) を基調とします。
    * **発光部:** 機体の動力源であり、アイデンティティとなる色は、鮮やかな**シアン** (`#00FFFF`) です。これはゲーム全体の「闇とネオン」というアートスタイルに準拠しており、暗い宇宙空間で極めて高い視認性を確保します。輪郭線、後述するエネルギーコア、そして噴射光がこの色で統一されます。

#### **アニメーションとエフェクト**
* **エネルギーコア:** 機体中央には、本機の心臓部である円形のエネルギーコアが埋め込まれています。このコアは常に「ドクン…ドクン…」と、生命の鼓動のようにゆっくりとシアン色に脈動します。移動速度が上がると、この脈動のテンポもわずかに速くなり、機体の稼働状態を直感的に伝えます。
* **推進エフェクト:** 機体後部のノズルからは、常にシアン色のエネルギーが**光の帯**として放出されます。静止時は短く揺らめくだけですが、移動速度が上がるにつれて長く、激しくなり、最大速度では機体長の数倍にも及ぶ光の尾を引きます。これはプレイヤーに速度感をフィードバックする最も重要な視覚要素です。
* **強化の視覚的反映:** プレイヤーがスキルを取得し機体を強化すると、そのアップグレードが統計データだけでなく、見た目にも反映されます。
    * 例1: 「攻撃速度アップ」を取得すると、主翼の先端が鋭角的に伸長し、より攻撃的なフォルムへと変化する。
    * 例2: 「ドローン追加」スキルを取得すると、主翼の下部に小型の兵装ポッドが物理的にマウントされる。
* **機動:** 移動時は機体が進行方向にわずかに傾き（バンク）、高速で旋回する戦闘機らしい挙動を見せます。常に最も近い敵に機首を向け続けるため、後退しながら前方の敵を撃つといった、戦術的でダイナミックな動きが生まれます。

#### **ゲームプレイ上の特性**
**攻撃・機動力特化型**というコンセプトは、ハイリスク・ハイリターンなプレイスタイルを推奨します。初期移動速度が高いため、敵の弾幕を素早く抜けたり、危険な状況から一気に離脱したりする「ヒット・アンド・アウェイ」戦法が得意です。反面、初期HPは低めに設定されているため、一度の操縦ミスが命取りになる可能性があります。プレイヤーの純粋な回避スキルと、的確な状況判断能力が試される、上級者向けの機体と言えます。

#### **Canvas描画ヒント**
* `beginPath()`と`lineTo()`でシャープな胴体と翼の基本パスを作成します。翼の優雅な曲線は`bezierCurveTo()`で表現可能です。
* エネルギーコアの脈動は、`Math.sin()`を利用して`shadowBlur`の値や円の半径を周期的に変化させることで実現します。
* 推進エフェクトの光の尾は、過去のフレームの機体位置を配列で記憶し、それらを繋ぐように半透明のグラデーションをかけたポリゴン（`fill()`）やライン（`stroke()`）を描画することで、滑らかな軌跡を表現できます。

### 10.1.2. 自律式ペットドローン "Pico-Rabbit" (詳細)

#### **コンセプト**
**「戦場を駆け回る、愛らしくもタフなマスコット」**
"Pico-Rabbit"は、その名の通りウサギをモチーフにした、親しみやすく可愛らしいデザインの機体です。プレイヤーには、危険な戦場で信頼できるペットや相棒と共にサバイバルするような、心温まる体験を提供します。攻撃的な"Strider-V"とは対照的に、その真価は直接的な戦闘力よりも、卓越した生存能力とサポート性能にあります。勝利の鍵は「敵を速く倒すこと」ではなく「誰よりも長く生き残ること」です。

#### **デザイン詳細**
* **基本形状:** 全体的に丸みを帯びた、柔らかく有機的なフォルムで構成されています。
    * **胴体 (Body):** `arc()`で描かれる、滑らかな卵型のカプセルのような形状。物理的な兵装は見当たらず、つるりとした表面が非戦闘的な印象を与えます。
    * **耳 (Ears):** 胴体の上部からは、本機最大の特徴である長大な**ウサギの耳**のようなパーツが2本伸びています。これらは単なる飾りではなく、周囲の情報を収集する多機能センサーであり、感情表現の役割も担います。
    * **顔 (Face):** 物理的な目や口はありません。代わりに、機体の正面に配置された円形のソフトライトが、状況に応じて色や明るさを変えることで、擬似的な「表情」を表現します。

* **基本カラー:**
    * **本体:** 柔らかく、優しい印象を与える**パステルピンク** (`#FFC0CB`) を基調とします。素材は硬い金属ではなく、滑らかなセラミックや高級プラスチックのような質感をイメージさせます。
    * **発光部:** 輪郭線や各種エフェクトには、純粋さや清潔感を象徴する**白** (`#FFFFFF`) を使用。ピンクのボディカラーとのコントラストが、機体の可愛らしさを一層引き立てます。

#### **アニメーションとエフェクト**
* **耳の揺れ:** "Pico-Rabbit"のチャームポイントです。移動時には、慣性によって長い耳が「ぴょこ、ぴょこ」と、少し遅れて追従するように揺れます。この物理ベースの自然な揺れが、生きているかのような生命感と愛らしさを生み出します。
* **攻撃エフェクト:** 攻撃はシャープなレーザーではなく、テーマに沿った遊び心のあるものです。敵に向かって、**星形やハート形の、ふんわりとした光弾**が発射されます。着弾音も「ポンッ」というような柔らかいサウンドにします。
* **被ダメージ表現:** HPが減少すると、機体の上に漫画のフキダシが表示され、その中にしょんぼりとした顔文字**「(´・ω・｀)」**が数秒間表示されます。これはプレイヤーにダメージを受けたことを明確に伝えつつ、機体への愛着を深めるための演出です。
* **待機モーション:** 移動を止めると、その場で「ぷかぷか」と穏やかに上下に浮遊します。耳もリラックスしたように少し垂れ下がり、戦闘の合間の癒やしとなります。

#### **ゲームプレイ上の特性**
**サポート・生存特化型**の性能は、特にゲームに不慣れなプレイヤーにとって心強い味方となります。初期HPが高く、敵がドロップする経験値ジェムやアイテムを自動で吸い寄せる範囲が広いため、プレイヤーは危険な敵集団に接近するリスクを冒さずとも、安定して成長できます。レベルアップ時には防御系（HP増加、ダメージカット）や補助系（移動速度、アイテム出現率アップ）のスキルが出現しやすく、プレイスタイルは自然と「耐えて、集めて、じっくり強くなる」方向へと導かれます。

#### **Canvas描画ヒント**
* `arc()`を多用して、丸みを帯びたボディを形成します。
* 耳の揺れるアニメーションは、機体の移動ベクトルと逆方向に少し力を加えるような物理演算か、より簡易的には`Math.sin()`を使って周期的な回転を加えることで、生き生きとした動きを表現できます。
* 被ダメージ時のフキダシは、`beginPath()`, `lineTo()`, `arcTo()`などを組み合わせて形状を描き、`fillText()`で内部に顔文字を描画します。フキダシは一定時間後にフェードアウトさせます。
* 星形やハート形の弾丸は、あらかじめ頂点座標を定義しておき、`lineTo()`でパスを繋いで`fill()`で塗りつぶすことで描画します。

#### 10.4. 背景デザイン (宇宙空間)

  * **コンセプト: 「静かなる躍動感」**
      * 背景は主役を引き立てる舞台装置。派手すぎず、しかし単調でもない、プレイヤーの動きによってダイナミックに変化する背景を目指します。
  * **レイヤー構造によるパララックス効果（多重スクロール）**
      * 背景を4つのレイヤーに分け、それぞれ異なる速度でスクロールさせることで、深い奥行きと強力な移動感を生み出します。
          * **レイヤー1 (最深層):** 遠景の星々（極低速）
          * **レイヤー2 (中間層):** 星雲 (Nebula)（低速）
          * **レイヤー3 (近景層):** 通過する星々（中速）
          * **レイヤー4 (最前景):** 宇宙塵/マイクロアステロイド（高速）
  * **境界線「デッドリー・ゾーン」のビジュアルデザイン**
      * **コンセプト:** 一目で「危険な場所」と分かる、SF的なエネルギーの壁。
      * **ビジュアル:** 半透明の赤いエネルギーの壁が不安定に揺らめき、内部で稲妻のようなプラズマが走る。プレイヤーが接触すると、機体との間に放電（アーク）が発生し、画面端が赤くフラッシュして危険を警告します。
