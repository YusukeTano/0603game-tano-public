# リズム・サバイバー (Rhythm Survivor)\n\n音楽とシンクロする、爽快リズムサバイバーゲーム。BGMのリズムに合わせて敵を倒し、パワーアップしていくことで得られる高揚感と、大量の敵から生き残るスリルを融合させた体験を提供する。\n\n## 🎮 ゲーム概要\n\n- **ジャンル**: 2D見下ろし視点のリズムアクション x サバイバーライクゲーム\n- **操作**: マウスで移動、左クリックで武器チェンジ、右クリックで必殺技\n- **目標**: 制限時間まで生き残る、または出現するボスを倒す\n\n## 🏗️ プロジェクト構造\n\n```\n0603game/\n├── src/                    # ソースコード\n│   ├── js/                # JavaScript モジュール\n│   │   ├── entities/      # ゲームエンティティ (プレイヤー、武器等)\n│   │   ├── audioManager.js # オーディオ管理システム\n│   │   ├── particleSystem.js # パーティクルシステム\n│   │   └── main.js        # メインゲームクラス\n│   ├── css/               # スタイルシート\n│   │   └── main.css       # メインスタイル\n│   ├── config/            # 設定ファイル\n│   │   └── gameConfig.js  # ゲーム設定\n│   ├── utils/             # ユーティリティ\n│   │   ├── collision.js   # 衝突検知\n│   │   └── objectPool.js  # オブジェクトプール\n│   └── assets/            # アセット（画像、音声）\n├── dist/                   # ビルド出力\n│   └── index.html         # メインHTMLファイル\n├── docs/                   # ドキュメント\n├── tests/                  # テストファイル\n├── scripts/                # ビルドスクリプト\n├── public/                 # 旧構造（互換性のため保持）\n└── .github/workflows/      # GitHub Actions\n```\n\n## 🚀 開発環境セットアップ\n\n### 必要な環境\n- Node.js (v16以上推奨)\n- Python 3 (開発サーバー用)\n\n### クイックスタート\n\n```bash\n# リポジトリをクローン\ngit clone <repository-url>\ncd 0603game\n\n# 開発サーバーを起動\nnpm run dev\n# または\npython3 -m http.server 8080\n\n# ブラウザで http://localhost:8080/dist/ を開く\n```\n\n### ビルド\n\n```bash\n# プロダクションビルド\nnpm run build\n\n# ビルド結果を確認\nnpm run serve\n```\n\n## 🎯 主要機能\n\n### 1. リズムコンボシステム\nBGMのビート（例えば4拍に1回など）のタイミングで経験値アイテムを回収するとコンボが発生。コンボが繋がるほど、取得経験値にボーナスが付いたり、一時的に移動速度がアップしたりする。\n\n### 2. スキルビルドと進化\nレベルアップ時に、攻撃範囲が広がるスキル、弾数を増やすスキル、移動速度を上げるパッシブスキルなど、様々なスキルをランダムに3つ提示。プレイヤーは一つを選んで自分だけのビルドを構築していく。\n\n### 3. フィーバーモード\nリズムコンボを一定数（例: 50コンボ）繋げると「フィーバーモード」に突入。一定時間、BGMがより激しい曲調に変化し、全ての攻撃が大幅に強化され、プレイヤーは無敵になる。\n\n### 4. 多彩なエネミーとボス\n最初はゆっくりと近づいてくるだけの敵から、弾を撃ってくる敵、素早い敵など、時間経過と共に様々な種類の敵が出現。\n\n### 5. 武器システム\n- ベーシック: バランスの取れた基本武器\n- ラピッド: 高い連射力、低いダメージ\n- ヘビー: 高いダメージ、遅い連射\n- スプレッド: 複数弾を同時発射\n- スナイパー: 長射程、高ダメージ\n- エクスプローシブ: 爆発ダメージ\n\n## 🎨 技術的特徴\n\n### モジュラー設計\n- ES6モジュールによるコード分割\n- 責任の分離（オーディオ、パーティクル、衝突検知等）\n- オブジェクトプールによるメモリ効率化\n\n### パフォーマンス最適化\n- 二乗距離計算による衝突検知高速化\n- パーティクルシステムの最適化\n- カメラベースの描画カリング\n\n### オーディオシステム\n- Web Audio API による動的音楽生成\n- レベルに応じた音楽の複雑化\n- フィーバーモード時の音楽変化\n\n## 🔧 設定\n\n### ゲーム設定\n`src/config/gameConfig.js` で以下を調整可能：\n- プレイヤー設定（HP、速度、サイズ等）\n- 敵の設定（種類別のステータス）\n- 武器の設定（ダメージ、射程、連射速度等）\n- ゲーム機能（コンボ、フィーバーモード等）\n\n### オーディオ設定\n- マスター音量\n- 音楽/効果音のバランス\n- テンポの変化率\n\n## 🚀 デプロイ\n\nプロジェクトは GitHub Actions により自動的に AWS S3 にデプロイされます。\n\n### 手動デプロイ\n```bash\n# ビルド\nnpm run build\n\n# AWS CLI でデプロイ (設定済みの場合)\naws s3 sync ./dist/ s3://your-bucket-name/ --delete\n```\n\n## 🤝 コントリビューション\n\n1. フォークしてブランチを作成\n2. 変更を実装\n3. テストを実行\n4. プルリクエストを作成\n\n## 📝 ライセンス\n\nMIT License\n\n## 🎵 アート・サウンドの方向性\n\n- **アート**: ネオンやサイバーパンク風の、シンプルで視認性の高いベクターアートスタイル\n- **サウンド**: BGMをゲームプレイの核とする。BPM（テンポ）が明確なエレクトロニック・ミュージックを採用し、フィーバーモードではさらにアップテンポな曲に切り替わる\n\n## 🔍 デバッグ情報\n\n開発時は以下の方法でデバッグ可能：\n\n```javascript\n// ブラウザコンソールで\nwindow.game.player.level = 10;  //
